# 一、AI 桌面助手完整技术构造文档

## 1. 项目概述

### 1.1 项目名称
命名为：
- **QuickAI Assistant**

### 1.2 项目目标
实现一个面向个人使用的 Windows 桌面 AI 助手，支持通过全局快捷键快速唤起，完成以下能力：

- 读取当前选中文本
- 截图并 OCR 识别文字
- 使用预设 Prompt 模板快速组织问题
- 调用用户自定义的大模型 API
- 可选联网检索并抓取网页正文，以提高回答准确率，需要在实现快速响应的基础上完成
- 将模型结果复制或回填到当前应用
- 最终打包为独立 exe 程序

### 1.3 产品定位
- 个人效率工具
- 常驻后台的轻量桌面应用
- 不依赖浏览器插件
- 不依赖复杂云端服务
- 用户自行配置模型与搜索 API

---

## 2. 核心使用场景

### 2.1 选中文本问答
用户在任意应用里选中一段内容，按快捷键后，工具自动读取选中文本，发给模型，返回解释、总结、翻译、润色等结果。

### 2.2 截图 OCR 问答
用户使用快捷键框选屏幕区域，系统截图后进行 OCR，自动提取文字，再交给模型处理。

### 2.3 模板化提问
用户在设置中提前配置 Prompt 模板，例如：
- 总结
- 翻译
- 润色
- 代码解释
- 会议纪要
- 问题排查

调用时只需选择模板或使用模板快捷键即可。

### 2.4 联网检索增强
当问题涉及事实、最新信息、版本、价格、政策、文档等内容时，用户可以选择是否触发联网检索：
- 先走搜索 API
- 再抓取网页正文
- 将检索结果注入 Prompt
- 提高回答准确率

### 2.5 快速结果回填
模型输出后，支持：
- 一键复制
- 自动粘贴到当前光标位置
- 保留在浮窗内继续修改

## 3. 总体架构

系统采用单机桌面架构，分为以下模块：

1. **UI 层**
2. **快捷键与系统集成层**
3. **输入采集层**
4. **OCR 层**
5. **Prompt 编排层**
6. **模型调用层**
7. **联网检索层**
8. **本地存储层**
9. **打包部署层**

## 4. 技术选型

### 4.1 开发语言
- **Python 3.11**

### 4.2 桌面 UI
- **PySide6 / Qt**

原因：
- 支持无边框悬浮窗
- 支持置顶和透明效果
- 适合做命令面板式交互
- 对系统托盘支持成熟

### 4.3 截图
- **mss**

原因：
- 轻量
- 速度快
- 简洁易用

### 4.4 OCR
- **PaddleOCR**

原因：
- 中文识别较强
- 适合截图 OCR
- 可以本地运行

### 4.5 模型调用
- **httpx** 或 **requests**
- 优先兼容 **OpenAI-compatible API**

### 4.6 联网检索
- **搜索 API**：Tavily / Bing Search API / SerpAPI 任选其一
- **网页正文抓取**：requests + BeautifulSoup4 + readability-lxml 或 trafilatura

### 4.7 本地存储
- **SQLite**

### 4.8 打包
- **PyInstaller**

## 5. 详细功能设计

## 5.1 全局快捷键

### 功能要求
- 任意应用中可触发
- 支持自定义
- 支持冲突检测
- 支持三类动作：
  1. 打开主窗口
  2. 读取选中文本
  3. 截图 OCR

### 推荐默认快捷键
- `Ctrl + Space`：打开主窗口
- `Ctrl + Shift + T`：读取选中文本
- `Ctrl + Shift + S`：截图 OCR

### 实现思路
- 程序后台常驻
- 注册系统全局热键
- 热键触发后直接调用对应服务

---

## 5.2 悬浮主窗口

### 窗口特点
- 无边框
- 置顶
- 可拖动
- 小尺寸默认隐藏
- 支持淡入淡出
- 支持半透明外观

### UI 布局
#### 顶部区域
- 模型选择
- 搜索开关
- 模板选择
- 设置入口

#### 中部区域
- 输入框
- OCR/取词结果
- 模型回答输出区

#### 底部区域
- 发送
- 复制
- 粘贴
- 清空
- 切换模板
- 继续追问

### 交互原则
- 默认尽量少打扰
- 输入后回车发送
- 回答支持流式输出
- 结果区可直接复制

---

## 5.3 选中文本读取

### 功能目标
读取用户当前在任意应用中的选中文本。

### 推荐实现方案
使用“剪贴板中转法”：
1. 保存当前剪贴板内容
2. 模拟 `Ctrl+C`
3. 读取剪贴板文本
4. 恢复原剪贴板

### 优点
- 通用性高
- 实现简单
- 对大多数应用有效

### 注意事项
- 某些受保护应用可能无法复制
- 需要增加短延迟等待剪贴板更新
- 需要恢复原剪贴板，避免打扰用户

---

## 5.4 截图 OCR

### 功能目标
通过快捷键框选屏幕区域，识别图片中的文字。

### 流程
1. 用户触发截图快捷键
2. 弹出半透明遮罩层
3. 用户拖拽选择区域
4. 获取截图
5. 调用 OCR 识别
6. 将识别结果填入输入框

### OCR 增强建议
- 识别前进行图像预处理：
  - 灰度化
  - 二值化
  - 放大
  - 去噪
- 识别后进行文本清洗：
  - 去除多余空格
  - 修复换行
  - 合并断行

---

## 5.5 Prompt 模板系统

### 目标
让用户为不同任务预先定义 Prompt，减少每次手写提示词的成本。

### 模板字段
每个模板建议至少包含：
- 模板 ID
- 名称
- 分类
- 系统提示词
- 用户提示词模板
- 是否启用联网检索
- 默认模型
- 输出格式要求
- 备注

### 示例模板
#### 1. 总结模板
- 系统提示词：你是一个严谨的总结助手
- 用户模板：请总结以下内容，并提炼重点：{input}

#### 2. 翻译模板
- 系统提示词：你是一个专业翻译助手
- 用户模板：请将以下内容翻译成中文：{input}

#### 3. 代码解释模板
- 系统提示词：你是资深软件工程师
- 用户模板：请解释以下代码的作用、关键逻辑和潜在问题：{input}

---

## 5.6 模型调用层

### 目标
支持用户自行配置大模型 API。

### 配置项
- API Base URL
- API Key
- 模型名
- 温度
- 最大输出长度
- 是否流式输出

### 支持方式
建议优先做 **OpenAI-compatible** 适配：
- OpenAI
- DeepSeek
- 通义千问兼容接口
- 部分第三方网关
- 私有部署兼容接口

### 调用流程
1. 拼接 system prompt
2. 拼接用户 prompt
3. 可选加入 OCR / 取词 / 搜索上下文
4. 发起流式请求
5. 实时更新 UI
6. 完成后保存会话

---

## 5.7 联网检索增强

### 目标
提升事实类、时效类、知识类问题的正确率。

### 设计原则
- 默认不开启或按需开启
- 用户可手动切换
- 可由模板控制是否启用
- 对高风险/高时效问题自动建议开启

### 检索策略
#### 第一步：搜索
调用搜索 API 获取前 3~5 条结果：
- 标题
- 摘要
- URL

#### 第二步：正文抓取
对重要结果抓正文：
- 使用 requests 拉取 HTML
- 通过 trafilatura / readability-lxml 提取正文
- 去掉导航、广告、脚本

#### 第三步：内容压缩
将正文裁剪、摘要化：
- 保留标题
- 保留关键段落
- 保留引用链接

#### 第四步：注入 Prompt
将检索结果作为参考上下文传给模型。

---

## 5.8 结果输出与回填

### 输出方式
1. 复制到剪贴板
2. 自动粘贴到当前应用
3. 仅在悬浮窗查看

### 推荐默认策略
- 默认“复制”
- 用户可选“复制并粘贴”

### 回填注意
- 自动粘贴前提示用户
- 避免误输入到敏感窗口
- 某些场景只复制不粘贴更稳妥

## 7. Prompt 编排规则

---

## 7.1 输入优先级
如果同时存在多种输入，按以下优先级处理：

1. 用户手动输入
2. OCR 识别文本
3. 选中文本
4. 剪贴板文本

---

## 7.2 Prompt 组成结构
统一拼接为：

```text
System Prompt
+ Template Prompt
+ User Input
+ OCR Text / Selection Text
+ Search Context
+ Output Constraints
```

---

## 7.3 上下文裁剪
为了控制 Token：
- 输入文本过长时自动截断
- 搜索结果最多保留前几条
- 正文仅保留关键段落
- 提供“展开全部”按钮

## 8. 联网检索实现细节

---

## 8.1 搜索 API 接口设计
建议统一成以下返回结构：

```json
{
  "query": "xxx",
  "results": [
    {
      "title": "标题1",
      "url": "https://...",
      "snippet": "摘要1"
    }
  ]
}
```

---

## 8.2 正文抓取流程
对每个候选链接：
1. 请求网页
2. 检查 content-type
3. 解析 HTML
4. 提取正文
5. 过滤重复内容
6. 生成简化摘要

---

## 8.3 抓取失败处理
如果抓正文失败：
- 退回搜索摘要
- 不阻塞主流程
- 继续回答

---

## 8.4 安全控制
- 不自动访问用户指定的敏感本地地址
- 仅处理标准 HTTP/HTTPS 页面
- 增加超时限制
- 限制最大抓取字节数

## 12. 关键实现  略

### 12.1 优先做最小闭环
先完成：
- 全局快捷键
- 手动输入
- 模板
- 模型调用
- 复制结果

然后再做：
- 取词
- OCR
- 联网检索

### 12.2 模型层与业务层解耦
以后换模型时不改流程代码，只改适配器。

### 12.3 联网检索作为可插拔能力
模板可配置是否启用联网，不强制所有问题检索。

### 12.4 识别失败要优雅降级
比如：
- OCR 失败时允许用户手改
- 搜索失败时直接回答，不中断

## 14. 风险与对策

### 14.1 快捷键冲突
对策：
- 允许用户修改
- 启动时检测冲突并提示

### 14.2 OCR 识别率不足
对策：
- 图像预处理
- PaddleOCR 优先
- 允许手动编辑文本

### 14.3 搜索结果噪声大
对策：
- 只取前几条
- 正文提取后再摘要
- Prompt 中要求引用来源

### 14.4 模型幻觉
对策：
- 联网检索增强
- 在 Prompt 中明确要求基于资料回答
- 输出中给出来源链接

### 14.5 打包体积偏大
对策：
- 先接受体积
- 后续再做资源优化

## AI 开发指令 Prompt

你是一名资深 Python 桌面应用工程师和架构师，请根据以下要求，为一个 Windows 个人使用的 AI 桌面助手项目生成可执行的代码实现方案，并直接开始按模块编写代码。

### 项目目标
实现一个通过全局快捷键唤起的 AI 助手，具备以下能力：
1. 读取当前选中文本
2. 截图并进行 OCR 识别
3. 支持模板化 Prompt
4. 支持用户自行配置大模型 API
5. 支持联网检索，并抓取网页正文作为上下文以提升回答准确率
6. 支持将模型输出快速复制或粘贴回当前应用
7. 最终能够使用 PyInstaller 封装为 Windows exe
8. 这是一个个人项目，要求轻量、可维护、实现成本低

### 技术栈要求
- Python 3.11
- PySide6
- PyInstaller
- mss 截图
- PaddleOCR OCR
- requests / httpx
- SQLite
- BeautifulSoup4 或 trafilatura 用于网页正文抓取
- 优先兼容 OpenAI-compatible API

### 架构要求
请按以下模块拆分：
- UI 层
- 全局快捷键管理
- 剪贴板/选中文本读取
- 截图服务
- OCR 服务
- Prompt 模板管理
- 大模型客户端
- 搜索 API 客户端
- 网页正文抓取器
- 会话与本地存储
- 设置管理
- 日志与错误处理

### 具体功能要求
1. 主窗口为无边框、置顶、可拖动、轻量的悬浮窗。
2. 支持全局快捷键：
   - 打开主窗口
   - 读取选中文本
   - 截图 OCR
3. 读取选中文本建议使用剪贴板中转法。
4. 截图 OCR 支持用户框选区域。
5. Prompt 模板支持预设、编辑、保存、分类。
6. 支持手动配置：
   - API Base URL
   - API Key
   - Model Name
