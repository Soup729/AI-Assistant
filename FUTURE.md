## 1.RAG功能（优先实现）

- 4.5已实现：在设置的‘增强功能’中新增超轻量的可选 RAG（本地笔记检索）模块。用户开启该功能后，需配置本地 Notes 文件夹路径以及用于文本向量化的云端 Embedding 模型与 API Key。软件启动时将挂载一个后台守护线程，采用专门针对 Markdown 语法的文本分块策略（如按标题层级或段落切分），在完全不阻塞前台操作的情况下，静默将 MD 文件切块、调用云端 API 向量化，并将原文落盘至 SQLite，向量数据存为 Numpy 本地索引文件。用户发起提问时，UI 需立即展示‘正在翻阅笔记...’的过渡动画以提升交互体验；底层同步将问题向量化，并直接使用 Numpy 执行余弦相似度计算，实现极轻量的‘向量 + 关键词’混合检索（Hybrid Search）。具体做法为：利用 SQLite 原生自带的 FTS5 虚拟表存储 Markdown 文本块并提供极速关键词全文检索，同时使用 Numpy 提供向量相似度检索。当用户提问时，双路并发召回结果，使用几行纯 Python 实现的 RRF（倒数排名融合）算法进行结果合并，取最终得分最高的 Top 3 段落，随后与预设语境 Prompt 组装并发送给大模型。该检索与组装链路必须进行极致的性能优化，以确保极低的响应延迟。生成结束后，需在回答底部追加渲染类似‘参考来源：[学习笔记.md]’的可点击标签，用户点击后直接调用 Windows 默认编辑器打开对应的本地源文件，实现高效的信息闭环。

## 2.性能优化

- 4.5已实现：专为 OCR 设计的“智能段落缝合”：OCR 识别出来的文本经常会有硬回车（换行符），比如一句话被截断成了两行。直接喂给 AI 会影响大模型的理解，还会多吃 Token。对此，写一个高效的正则表达式后处理函数。规则：如果一行的结尾不是标点符号（句号、问号、叹号、逗号等），且下一行有连贯字符，则自动去掉换行符，补上一个空格或直接拼接。（自由发挥补充更多可行、好用、高效的规则）
- 4.5已实现：全链路异步化：引入 qasync 库（将 Python 的 asyncio 与 Qt 的事件循环完美结合）。把所有网络请求（httpx.AsyncClient）、本地磁盘读写、RAG 向量计算全部用 async/await 包装，确保主 UI 线程达到丝滑的 60fps。

- 已修复：回填功能修复：当前使用回填时，还是未能真正实现回填效果，找到问题并修复，；
- 未实现：“快捷键劫持”实现的就地翻译/润色（无 UI 模式）：在设置内可选择开不开启，同时增加一个快捷键用于选择开不开启该功能。常规使用流程是：选中文本 -> 唤醒助手 -> 发送文本 -> 等待 -> 回填。而该功能做一个极速静默模式。用户选中文本后，按下比如读取选中文本的快捷键，程序不弹任何主窗口。只在鼠标旁边或者系统托盘闪过一个微小的加载动画。后台静默调用大模型进行润色。并且实现流式直接回填，AI 一边生成，一边模拟键盘在鼠标选中处像打字一样把结果敲出来，没有任何多余的 UI 干扰。
- 未实现：结果渲染-轻量级 Markdown 渲染：使用 Python 的 mistune 库，在收到流式（Streaming）返回的文本时，将其转换为基础的 HTML，然后渲染在轻量级的 QTextBrowser 中。自己写点简单的 CSS，实现代码块高亮（可借助 pygments）和表格渲染，体验极佳且体积几乎零增加；
- 未实现：智能富文本注入（Smart Clipboard HTML 注入）：大模型经常输出带有Markdown 表格（| Name | Age |）或加粗、标题格式的内容。如果直接模拟 Ctrl+V，粘贴进 Word、Excel 或邮件里，只会是一堆难看的纯文本代码，用户还得手动排版。对此，使用 markdown 库将 AI 的输出转换为 HTML，使用 win32clipboard，将剪贴板的格式强行设置为 HTML Format（这是 Windows 剪贴板的一种标准格式，包含特定的 header）。效果：当 AI 帮你总结了一份表格数据并自动回填时，粘贴到 Excel 里直接就是原生的 Excel 表格单元格；粘贴到 Word 里直接就是带字号和加粗排版的精美内容。这绝对是让用户惊呼“卧槽”的杀手级体验。
