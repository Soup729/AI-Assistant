# 未来优化

## 1.RAG功能（优先实现）

- 在设置的‘增强功能’中新增超轻量的可选 RAG（本地笔记检索）模块。用户开启该功能后，需配置本地 Notes 文件夹路径以及用于文本向量化的云端 Embedding 模型与 API Key。软件启动时将挂载一个后台守护线程，采用专门针对 Markdown 语法的文本分块策略（如按标题层级或段落切分），在完全不阻塞前台操作的情况下，静默将 MD 文件切块、调用云端 API 向量化，并将原文落盘至 SQLite，向量数据存为 Numpy 本地索引文件。用户发起提问时，UI 需立即展示‘正在翻阅笔记...’的过渡动画以提升交互体验；底层同步将问题向量化，并直接使用 Numpy 执行余弦相似度计算，实现极轻量的‘向量 + 关键词’混合检索（Hybrid Search）。具体做法为：利用 SQLite 原生自带的 FTS5 虚拟表存储 Markdown 文本块并提供极速关键词全文检索，同时使用 Numpy 提供向量相似度检索。当用户提问时，双路并发召回结果，使用几行纯 Python 实现的 RRF（倒数排名融合）算法进行结果合并，取最终得分最高的 Top 3 段落，随后与预设语境 Prompt 组装并发送给大模型。该检索与组装链路必须进行极致的性能优化，以确保极低的响应延迟。生成结束后，需在回答底部追加渲染类似‘参考来源：[学习笔记.md]’的可点击标签，用户点击后直接调用 Windows 默认编辑器打开对应的本地源文件，实现高效的信息闭环。

## 2.性能优化

- ！专为 OCR 设计的“智能段落缝合”：OCR 识别出来的文本经常会有硬回车（换行符），比如一句话被截断成了两行。直接喂给 AI 会影响大模型的理解，还会多吃 Token。对此，写一个高效的正则表达式后处理函数。规则：如果一行的结尾不是标点符号（句号、问号、叹号、逗号等），且下一行有连贯字符，则自动去掉换行符，补上一个空格或直接拼接。（自由发挥补充更多可行、好用、高效的规则）
- 模块懒加载与后台预热：主程序启动时，只加载 PySide6 和全局快捷键监听模块，瞬间完成系统托盘挂载；UI 启动完成后，开启一个低优先级的后台线程，静默加载 RapidOCR 和 sherpa-onnx 引擎到内存中；像 RAG 检索引擎、大模型网络请求库等，不要在文件头部 import，而是在用户实际触发该功能时的函数内部进行 import；
- 内存按需释放：设计一个高效超时回收机制。比如检测到超过 30 分钟未调用截图 OCR 或语音识别，自动释放 engine 实例并调用 gc.collect()。下次使用时再重新加载（仅牺牲第一次调用的几百毫秒）。
- 全链路异步化：强烈建议引入 qasync 库（将 Python 的 asyncio 与 Qt 的事件循环完美结合）。把所有网络请求（httpx.AsyncClient）、本地磁盘读写、RAG 向量计算全部用 async/await 包装，确保主 UI 线程达到丝滑的 60fps。

## 3.用户体验优化

- ！回填功能修复：当前使用回填时，程序显示找不到回填窗口，找到问题并修复；
- ！窗口无焦点模式（Toast / 浮窗体验）：有时用户只想选中文本，按快捷键让 AI 翻译，结果主窗口弹出来，抢走了当前工作软件（如 Word/代码编辑器）的输入焦点，打断了用户的心流。对此，在 PySide6 中，对于“仅展示结果”或“流式输出翻译”的悬浮窗，设置 Qt.WindowStaysOnTopHint | Qt.WindowDoesNotAcceptFocus（对应 Windows 底层的 WS_EX_NOACTIVATE 属性）。这样窗口会置顶显示结果，但不会抢夺用户的键盘焦点，用户可以边看 AI 的翻译边继续在原软件里打字。
- ！“快捷键劫持”实现的就地翻译/润色（无 UI 模式）：在设置内可选择开不开启，同时增加一个快捷键用于选择开不开启该功能。常规使用流程是：选中文本 -> 唤醒助手 -> 发送文本 -> 等待 -> 回填。而该功能做一个极速静默模式。用户选中文本后，按下比如读取选中文本的快捷键，程序不弹任何主窗口。只在鼠标旁边或者系统托盘闪过一个微小的加载动画。后台静默调用大模型进行润色。完成后，直接模拟 Ctrl+V 替换掉用户刚才选中的文本。体验：用户选中一段烂文章，按个快捷键，等一秒钟，文章就在眼前自动变成了优美的文字，没有任何多余的 UI 干扰。
- ！结果渲染-轻量级 Markdown 渲染：使用 Python 的 mistune 库，在收到流式（Streaming）返回的文本时，将其转换为基础的 HTML，然后渲染在轻量级的 QTextBrowser 中。自己写点简单的 CSS，实现代码块高亮（可借助 pygments）和表格渲染，体验极佳且体积几乎零增加；
- ！智能富文本注入（Smart Clipboard HTML 注入）：大模型经常输出带有Markdown 表格（| Name | Age |）或加粗、标题格式的内容。如果直接模拟 Ctrl+V，粘贴进 Word、Excel 或邮件里，只会是一堆难看的纯文本代码，用户还得手动排版。对此，使用 markdown 库将 AI 的输出转换为 HTML，使用 win32clipboard，将剪贴板的格式强行设置为 HTML Format（这是 Windows 剪贴板的一种标准格式，包含特定的 header）。效果：当 AI 帮你总结了一份表格数据并自动回填时，粘贴到 Excel 里直接就是原生的 Excel 表格单元格；粘贴到 Word 里直接就是带字号和加粗排版的精美内容。这绝对是让用户惊呼“卧槽”的杀手级体验。
- 输出中断与“流式回填”：提供一个明确的 停止生成 (Esc) 按钮或快捷键，直接 cancel 掉网络请求；允许用户开启“流式直接回填”模式。AI 一边生成，助手一边模拟键盘在目标软件里像打字一样把结果敲出来，这在视觉上极具“AI 替我工作”的震撼感。

## 4. 架构进一步扩展

- 插件化系统：利用 Python 的 importlib 动态加载机制，设计一个简单的 plugins/ 目录，比如：OCR 是一个插件，ASR 是一个插件，甚至连“翻译选中文本”也是一个插件。
- 便捷的模型配置与测速：在设置界面加一个 “网络连通性测试 (Ping)” 和 “模型能力测试” 按钮。填完直接测延迟，如果失败，在 UI 上给出极其明确的报错翻译（比如：“API Key 余额不足”或“代理未设置”），大幅降低用户的上手门槛。
- 增加目录模式打包的教程：使用 --onedir（目录模式）打包，生成一个包含 app.exe 和一堆 .dll、.pyd 的文件夹。目录模式下的启动速度是毫秒级的。分发时，使用 Inno Setup（一个免费的 Windows 安装包制作工具）把这个文件夹打包成一个标准的 Setup.exe。不仅显得专业，还能自动创建桌面快捷方式、注册开机自启、甚至直接帮你处理写入注册表的全局快捷键权限。
